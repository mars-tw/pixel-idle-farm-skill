# 像素農場內容擴充路線圖 R1

- **專案**：pixel-idle-farm-skill（《阿軒割割陽光農場》PWA / vanilla JS）
- **審查角色**：資深遊戲設計師＋敘事總監
- **範圍**：內容豐富度、故事性、耐玩度 — **只規劃、不改程式碼**
- **審查基準（真實檔案）**：
  - `src/config.js` — 作物、建築、動物、任務、信件、NPC、季節天氣、成就
  - `src/game.js` — 經濟倍率、訂單生成、信件解鎖、故事推進、建築封頂
  - `src/state.js` — 存檔結構、`story` / `mail` / `flags` / `collections`
  - `src/ui.js` — 故事面板、信箱 UI、章節進度顯示
- **重要註記**：專案**沒有** `src/rules.js`；規則集中在 `game.js`（與 `docs/GROK_REVIEW_farm_R2.md` 一致）。
- **約束前提**（所有擴充必須遵守）：
  1. 維持 vanilla JS + 本地存檔，無後端。
  2. 尊重現有**經濟封頂**：建築 `maxCount`、成長光環連乘上限、訂單契約價、離線 8h、連單 +100%、成就每枚 +2% 售價等；新內容以**橫向深度**為主，避免再開可堆疊的永久成長倍增。
  3. 佔位／atlas 美術：新作物優先用 emoji 後備或既有 `crops` / `crops2` / `crops3` 空位；新建築優先重用 `buildings` / `structures-nature` / props；新角色優先重用 `npcs-48x64` 色板變體或對話-only。
  4. 資料驅動優先：新內容先落在 `config.js` 表，行為接既有 `evaluateLetters` / `QUESTS` / `NPC_SIDE_QUESTS` / `availableOrderItems` 管線。

## 0. 執行摘要

| 面向 | 現況評分（1–5） | 一句話 |
|------|----------------:|--------|
| 系統完整度 | 4.0 | 放置農場核心 + 地圖 RPG 骨架已齊：作物、建築、動物品質、季節天氣、訂單、章節任務、信件。 |
| 內容辨識度 | 2.5 | 作物／建築多為「更貴、更久、略不同季節標籤」；角色與地點有名字，但差異化不足。 |
| 敘事厚度 | 2.5 | 「回來接手祖母農場 → 修橋 → 照護 → 豐年祭 → 讀信回信」弧線清楚，但角色關係薄、結局後無第二弧。 |
| 中後期耐玩 | 2.0 | Ch5 回信後主軸結束；剩餘是訂單輪轉 + NPC 冷卻委託 + 採集 CD，缺少新目標曲線。 |

## 2. 敘事盤點：多薄弱？

### 2.1 已有的弧線（可用）

```
回歸荒田 → 第一桶金 → 清路修橋 → 東林貨源 → 動物親密度
→ 四季物產與鴨 → 豐年祭 → 祖母八封信 → 玩家回信（情感收束）
```

這是一條**完整但短**的「繼承與復耕」故事：主題清楚（土地需要被繼續照顧），信件文筆品質已高於一般 idle 文案。

### 2.2 薄弱點（設計層）

| 問題 | 現象 | 對體驗的影響 |
|------|------|--------------|
| **角色是看板** | NPC 台詞 2–3 句／phase，無記憶、無好感、無專屬結局 | 交付委託像「印鈔機」而非「人」 |
| **第五章是終章牆** | 讀信＋回信後 `questId=null`，無 Ch6 | 最有情感的時刻後立刻掉回純迴圈 |
| **信件單向** | 僅祖母 → 玩家；回信不觸發回信／鎮民反應 | 情感高潮不可延續 |
| **選擇為零** | 無對話選項、無路線分歧 | 重玩＝重走同一教學 |
| **地名與人設未落地玩法** | 「旱災」「祖父搭橋」「公告欄是舊農場木板」只在 lore 字串 | 世界觀像附錄，不是玩的理由 |
| **主角名斷裂** | Miri 美術 vs 文案「孩子」 | 認同感弱 |
| **助手無敘事** | 機器人升級像 UI 功能 | 錯失「請來幫手」的人味 |
| **節慶名存實亡** | `festival` 是訂單稀有度＋一建築，非時間事件 | 豐年祭無「到來感」 |

**敘事厚度評語**：骨架 3 分、角色 2 分、結局後續 1 分。不需要推倒重寫世界觀，而需要在**同一晨光鎮**上加「人與季節的第二季劇本」。

## 3. 「淺」處診斷（耐玩／內容）

### 3.1 作物淺

- 中後期 6 種季節作物本質是 **價格／時間階梯**，缺少：互斥輪作、加工鏈、角色偏好、節慶限定。
- 非季節作物（麥、蘿蔔、番茄、草莓、南瓜）在 Lv6 後無「當季身份」，卻仍是訂單 bulk 主力 → 季節系統感被稀釋。
- 溫室建築**不**解鎖跨季種植特權（文案寫控溫，效果只是 growthAura）→ 敘事與機制落差。

### 3.2 建築淺

- 三座成長光環（堆肥／蜂／溫室）玩法同族，差異僅數值。
- 筒倉 vs `storageLevel` 升級功能重疊。
- 無「社交建築」（茶室、工坊、展示架）可掛敘事與非成長獎勵。

### 3.3 放置後期無目標

Ch4–5 完成後，玩家擁有：滿農地曲線、助手、全作物、全動物路徑、採集點。  
剩餘目標僅：**刷金幣補滿升級**、**刷訂單 streak**、**刷親密度保 premium**、**等 CD 採集**。  
缺：章節後任務、聲望階、圖鑑完成度、節慶日曆、第二地區身份。

### 3.4 解鎖曲線斷層

| 時段 | 狀態 |
|------|------|
| 0–15 分 | 優秀：序章教學密、回饋快 |
| 15–60 分 | 良好：橋、東林、動物品質 |
| 1–3 小時 | 一般：四季＋鴨＋祭典單 |
| 之後 | 弱：等級 9–10、金幣堆疊、無新敘事門 |

### 3.5 敘事斷裂點

1. 修橋後「祖母舊物」暗示 → 實際獎勵主要是金幣／木材，無物件敘事。  
2. 老農「出師」後無師徒線。  
3. 豐年祭完成 → 立刻進「讀舊信」而非「祭典當週日常」。  
4. 回信結束 → 鎮民 `ch5done` 台詞封箱，無新 phase。

### 3.6 重玩無新意

- 無 New Game+、無聲望重置、無多農場、無隨機事件種子展示。
- 成就不開新內容，只加售價（且有上限感：10×2%=20%）。

## 4. 擴充原則（落地守則）

1. **先橫向、後縱向**：寧可多 1 條角色線，也不要再加第 4 個 growthAura。
2. **獎勵用「獨特」而非「更強」**：解鎖台詞、信件、裝飾、圖鑑條目、節慶訂單池、一次性收藏；金幣／XP 維持現有量級。
3. **接現有管線**：
   - 信件 → `LETTERS` + `letterUnlockSatisfied` 類型擴充
   - 任務 → `QUESTS` / `CHAPTER*_QUESTS` 或獨立 `CHAPTER6_*`
   - 角色 → `NPC_SIDE_QUESTS.steps` 延長或 `NPC_RELATION`
   - 物品 → `CROPS` / `FORAGE_ITEMS` / `COLLECTIBLES` + `availableOrderItems`
4. **atlas 策略**：P0 優先 emoji／現有 frame；P1 才規劃新 atlas 列；所有新 sprite 進驗證腳本前可用文字 UI。
5. **不破壞封頂**：新建築預設 `maxCount: 1`；避免全局 sellMul／growthMul 永久疊加；若需加成，用「當日／當季／活動窗」限時。

## 9. 經濟與平衡護欄（內容擴充必讀）

1. **新永久 sell/growth 倍率**：預設拒絕；若有，必須 `maxCount:1` 且可審計上限。  
2. **新訂單 payMult**：不超過現有 `festival 2.8`；活動只改權重不改公式上限。  
3. **關係／節慶獎勵**：優先 XP、收藏、外觀、限時便利（澆水、臨時訂單槽），少給裸金幣。  
4. **新作物期望金／時**：落在相鄰 unlock 作物的 ±15% 帶，避免廢除舊作物。  
5. **動物產能**：內容規劃**不**建議再加可蓋無限 coop；若未來加動物，先定 `maxCount` 與全場 cap。  
6. **成就售價**：已有最多約 +20%；P2 新成就改稱號，避免無上限爬升。  
7. **離線 8h**：任何新 timer（果樹、節慶）需定義 offline 行為與 cap。

## 13. 結語

本遊戲的系統完成度已超過敘事與中後期目標的完成度。  
**最快的耐玩增益**不是再加一種「更貴的南瓜」，而是：

1. 讓**季節**有訂單與事件指紋；  
2. 讓**四位鎮民**在支線後仍有信與心；
